sorter
通过设置某列sorter为true即可完成服务器排序，本地排序直接使用sorter即可，同时可以配置sorterFieldName和sorterOrderName以及sorterField配置服务器自定义的接收字段

<template>
    <div class="vp-raw">
        <w-table url="/table/page" :sort-direction="['up','down']" :headers="headers" />
    </div>
</template> 
<script lang="ts" setup>
const headers = [
    {
        dataIndex: "name",
        title: "姓名"
    },
    {
        dataIndex: "age",
        title: "年龄",
        sorter: true,
        sorterFieldName: "sortCode",
        sorterField:"AGE_NUMBER",
        sorterOrderName: "sortOrder"
    },
    {
        dataIndex: "address",
        title: "地址"
    },
    {
        dataIndex: "date",
        title: "加入时间"
    }
]
</script>

对于搜索的时候，有时候headers中的dataIndex和title，并不符合搜素表单的显示名称和请求参数，那么这个时候可以设置form属性中的itemAttrs中的name和label，name作为请求参数，label作为显示名称来覆盖表单的显示名称和请求参数，注意这里只会影响form-search，而不会影响比如table的搜索条件的插槽等

            type: "input",
            itemAttrs: {
                label: "名称",

        search: true,

<template>
    <div class="vp-raw">
        <w-table
            ref="tableRef"
            v-model:selectedRowKeys="selectionRowKeys"
            v-model:selectedRows="selectedRows"
            url="/table/page"
            :headers="headers"
            row-key="name"
            selection
        >
            <template #table-bar-action>
                <a-button @click="getRowKeys"> 获取以keys选中的数据</a-button>
                <a-button @click="getRows"> 获取行选中数据</a-button>
            </template>
        </w-table>
    </div>
</template>
<script lang="ts" setup>
import { Modal } from "ant-design-vue";
import { ref } from "vue";
const headers = ref([
    {
        dataIndex: "name",
        title: "姓名",
        form: {
            type: "input"
        }
    },
    {
        dataIndex: "teacher",
        title: "班主任"
    },
    {
        dataIndex: "address",
        title: "地址"
    },
    {
        dataIndex: "date",
        title: "加入时间"
    }
]);
const selectionRowKeys = ref([]);
const selectedRows = ref([]);
const getRowKeys = () => {
    Modal.info({
        title: "提示",
        content: JSON.stringify(selectionRowKeys.value)
    });
};

const tableRef = ref();
const getRows = () => {
    Modal.info({
        title: "提示",
        content: JSON.stringify(selectedRows.value)
    });
};
</script>

参数说明类型默认值
headers表头集合，扩展参数以及columnArray[]

url动态请求表格数据的url地址String
method请求方法，post|get 两种Stringpost

rowKey对于selection以及拖拽排序是很有必要。作用于虚拟dom重新渲染Stringid

resize开启表格高度自适应，一般用于表头固定，开启会消耗部分表格性能，而且w-table组件父容器必须设置高度，固定或者100% 都可以Booleanfalse
selection是否开启选择操作Booleanfalse
v-model (selectedRowKeys)双向绑定设置选中的行，以 row-key 作为标识Array[]

rowSelectionrowSelection)重写，覆盖了fixed、selectedRowKeys以及onChange三个属性，所以支持除了以上三个属性的所有属性Object

actionTitle操作栏标题String操作

showTitleBar是否显示整个·title（包含左标题和右按钮）Booleantrue

form查询条件表单属性设置。参考form-search 整体属性，不过这里剔除items和model属性。Object{}
pagination是否开启分页Booleantrue
v-model(page)页码，可双向绑定number1

loadAuto是否启用自动加载表格数据booleantrue
